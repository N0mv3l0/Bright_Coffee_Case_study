
# ☕ Bright Coffee Shop Sales Analysis

## 📖 Project Overview

The **Bright Coffee Shop Sales Analysis** is an end-to-end Business Intelligence and Data Analytics case study completed as part of the **BrightLearn Data Science & Analytics Program**. The project focuses on analyzing historical coffee shop sales data to generate actionable business insights for the newly appointed CEO of Bright Coffee Shop.

As a **Junior Data Analyst**, the objective was to transform raw transactional data into meaningful insights through data cleaning, SQL analysis, dashboard development, and executive reporting. The final solution enables management to understand sales performance, product trends, store performance, and customer purchasing patterns to support data-driven decision-making.

This project demonstrates the complete analytics workflow, from project planning and data architecture to interactive dashboards and an AI-powered Data Agent.



# 🎯 Business Problem

Bright Coffee Shop has recently appointed a new CEO whose goal is to increase revenue and improve product performance across its stores. Management requires a comprehensive sales analysis to answer key business questions, including:

* Which products generate the most revenue?
* Which store locations perform the best?
* What time periods generate the highest sales?
* Which products should be promoted or restocked?
* What recommendations can improve future sales performance?

The objective of this project was to provide these answers using historical sales data and visual analytics.



# 📌 Project Objectives

* Design the project architecture and analytics workflow.
* Clean and transform raw sales data using SQL.
* Calculate key business performance metrics.
* Analyze sales trends across stores, products, and time periods.
* Develop interactive dashboards for executive reporting.
* Present actionable recommendations supported by data.
* Build an AI-powered Data Agent capable of answering business questions using natural language.

# 🗂️ Project Workflow

The project followed the analytics lifecycle recommended in the case study.

A project roadmap was created using **Canva** and **Miro**, including:

* Business problem definition
* Data flow diagram
* Analytics workflow
* ETL process
* Dashboard planning
* KPI identification

## 2. Data Processing (Databricks)

The raw dataset was imported into Databricks where SQL was used to prepare the data for analysis.

### Data Transformations

* Cleaned and standardized the dataset
* Removed duplicate records
* Converted data types where necessary
* Calculated Total Revenue
* Created Month and Day fields
* Created Weekday and Weekend classifications
* Consolidated similar tea products into a single reporting category
* Generated summary tables for dashboard reporting

# 💡 Key Insights

The analysis produced several valuable business insights:

* Coffee beverages generated the largest share of overall revenue, making them the primary revenue drivers.
* Revenue performance differed across the three store locations, highlighting opportunities to replicate successful operational practices across stores.
* A small number of product types contributed a significant proportion of total sales, suggesting that inventory and promotional strategies should prioritize these products.
* Sales patterns varied across weekdays and weekends, indicating opportunities to optimize staffing schedules and promotional campaigns.
* Monthly sales trends revealed fluctuations in customer demand, providing management with valuable information for seasonal planning and inventory management.
* Grouping similar tea products into a single reporting category simplified product analysis and improved reporting consistency.
